# Document Style Guide

이 문서는 각 Publisher 에이전트가 참조하는 문서 스타일 가이드입니다.
새로운 sample 문서가 제공되면 해당 섹션을 업데이트합니다.

***

## BlogPublisher Style:

### 기본 원칙

* 한국어 작성
* 친근하고 읽기 쉬운 대화체
* 문단을 짧게 나누고 소제목을 적극 사용
* 핵심 키워드 굵게 강조
* **이모지 사용 금지** (예:🦊, 😊, 💪 등)

### 구조

1. **제목**: 호기심을 유발하는 제목 (30자 이내)
2. **부제목**: 본문 내용을 한 줄로 요약
3. **3줄 요약**: 핵심 내용을 3줄로 요약 (각 줄은 한 문장)
4. **본문**:
   * 도입: 독자의 관심을 끌 수 있는 질문이나 상황 제시
   * 전개: 핵심 내용을 소제목별로 정리
   * 마무리: 핵심 메시지 재강조 및 행동 유도(CTA)
5. **이미지**: 각 소제목 아래 관련 이미지 배치

### 제목 패턴

* **문제+해결형**: "\[기술] \[문제상황] 해결하기"
* **경험/후기형**: "\[작업명] 경험기 / 도입 후기"
* **질문형**: "\[개념], 어떻게 \[적용]하지?"
* **콜론형**: "\[주제]: \[부연 설명 또는 핵심 포인트]"
* **시리즈형**: "\[주제] 1편 / 2편" (다부작 구성 시)

### 기술 설명 요건

* 다이어그램/아키텍처 이미지: 복잡한 구조 설명 시 필수
* 수치 데이터: 표 또는 비교 리스트로 정리
* 참고 자료: 글 하단에 출처 링크 제공

### 톤 & 매너

* "\~입니다" 보다 "\~에요/\~해요" 스타일 선호
* 독자에게 말을 거는 듯한 문체
* 전문 용어는 쉽게 풀어서 설명
* 문단 길이: 3-5줄 이내
* 전문 용어는 첫 등장 시 풀어서 설명

### 포맷 요건

* docx + html 두 가지 포맷 출력
* html은 blodat 웹페이지 호환 필수
